Git の基本概念

リポジトリ (Repository)

プロジェクトのファイルとその変更履歴を保存する場所です。ローカルリポジトリとリモートリポジトリがあります。

コミット (Commit)

変更内容をリポジトリに記録することです。各コミットには一意のIDが付与されます。

ブランチ (Branch)

独立した作業ラインを作成する機能です。メインの開発ラインから分岐して作業できます。

Git の主要コマンド

git clone

リモートリポジトリをローカルにコピーします。

git branch

ブランチを作成・確認します。

git checkout

別のブランチに切り替えます。

git add

変更をステージングエリアに追加します。

git commit

ステージングされた変更をリポジトリに記録します。

git push

ローカルの変更をリモートリポジトリに送信します。

git pull

リモートリポジトリの変更をローカルに取り込みます。

GitHub ワークフロー

  1. Issue を作成して作業内容を明確にする
  2. 新しいブランチを作成する
  3. 変更を実装する
  4. 変更をコミットする
  5. リモートリポジトリにプッシュする
  6. Pull Request を作成する
  7. レビューを受ける
  8. マージする

SourceTree の使い方

SourceTree は Git の操作をグラフィカルに行えるツールです。

リポジトリの管理

複数のリポジトリを一元管理できます。

変更の可視化

変更内容を視覚的に確認できます。

ブランチの操作

ブランチの作成・切り替え・マージなどを簡単に行えます。
